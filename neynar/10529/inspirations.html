Twenty in-app browser / mini-app chrome references surveyed for the Neynar redesign. The eight archetypes on the main page are distilled from these. Scroll horizontally inside each section.

Source: NEYN-10529 · execution: CLIENTS-38

Crypto wallets mini-app + dapp browsers — scroll →

no shot yet
Rainbow ↗

Tall multi-row chrome: URL/search field with tab-switcher, plus a horizontal trending-dapps strip above content. Browser as destination, not transit.

Anti-reference for Neynar — leans into chunky because the browser is the value.

no shot yet
MetaMask ↗

Two-row: account/network selector pill on top, URL/search bar with tabs + overflow below. Connection state and chain surfaced inline rather than hidden.

Density-rich; example of how much state a wallet thinks it has to surface.

no shot yet
Coinbase Wallet ↗

Single-row: URL/title + padlock + tabs + overflow. Tabs treated like Safari (separate carousel). Chain switching silent.

Closest ancestor to Base App's chrome.

no shot yet
Trust Wallet ↗

Standard browser-like: back arrow, URL/title, share, overflow. Multichain selector behind the menu. Generic.

Neutral baseline.

no shot yet
Phantom ↗

Compact single-row: URL/title centered, × on left, share/⋯ on right. Network indicator implicit — no chain pill in chrome. Among the lightest.

Strong reference for "minimal wallet chrome" — closest to where Neynar likely wants to land.

no shot yet
Zerion ↗

URL/search bar with active-wallet identicon on left for fast multi-account switching. Identity is the leftmost affordance, not back.

Putting identity (not navigation) first is a Farcaster-shaped idea worth borrowing.

no shot yet
Uniswap Wallet ↗

Lean single-row: URL or dapp title, network pill, close. Dapp browser feels like a sub-mode rather than peer surface.

Good "browser as guest mode" reference.

no shot yet
Base App ↗

Mini-app surface: top header with app name + author + ×. Mini-app chrome is minimal but the embedding feed surface adds a thick parent header above it — net visual weight is high.

Direct competitor surface — exactly the chrome Neynar is being compared against.

Social apps in-app browsers — scroll →

no shot yet
X (Twitter) ↗

Top: done/close + site title + URL stacked + share + ⋯. Bottom: back/forward/share/Safari-handoff. Two-bar layout — chunky.

Anti-pattern: two bars where one would do.

no shot yet
Instagram ↗

Tall double-bar: × + stacked title/URL + open-in-Safari + ⋯ on top; back/forward/share on bottom. Famously chunky — eats ~120pt before any content shows.

The canonical anti-pattern. Pin to the moodboard as "do not do this."

no shot yet
TikTok ↗

Single top bar: × + truncated title (max 10–20 chars) + ⋯. Compact but truncation is aggressive.

Solves chunkiness by truncating hard. Tradeoff: title legibility.

no shot yet
Snapchat ↗

Branded yellow accents: × + title + share + ⋯ on top; back/forward + "open in Safari" on bottom. Less heavy than Instagram but still two-bar.

no shot yet
Reddit ↗

Single top bar + sticky "view comments" bar that returns to thread context — distinctive contextual return path.

"Return to context" affordance — mini-apps could borrow this for "return to cast."

no shot yet
Discord ↗

Stock WKWebView wrapper on iOS: close + URL + share + open-in-Safari; minimal Discord branding. Generic.

Reference for "lazy webview wrapper" — what NOT to ship if chrome is meant to feel native.

no shot yet
Warpcast ↗

Per spec: header above the mini-app showing app name + author + ×. Rendered header stacks below Warpcast's nav, making total chrome ~2 bars. The "too chunky" surface the redesign is targeting.

The thing being beaten. Dual-header is the core problem.

Mini-app paradigms canonical references — scroll →

no shot yet
Telegram Mini Apps ↗

Default: thin Telegram header with app title + back/close + ⋯. 2.0 introduced full-screen mode that hides BOTH Telegram top and bottom bars — most aggressive minimization.

Strongest reference. Full-screen escape hatch is exactly the lever Neynar should consider.

no shot yet
WeChat Mini Programs ↗

Iconic "capsule" menu: floating pill in the top-right with two buttons — ⋯ + ○. No URL bar. Title is the page title set by the mini-program.

The OG. Capsule pattern is probably the highest-leverage reference for Neynar — minimum chrome, maximum app surface.

no shot yet
LINE Mini Apps ↗

Platform-generated header with mandatory action button (cannot be hidden by developer); shows domain (unverified) or App Name + verified badge (verified). Redesigned late 2024 — colors no longer customizable.

"Platform owns the chrome, app cannot override" — a stance Neynar/Farcaster could take.

System primitives baselines — scroll →

no shot yet
SFSafariViewController (iOS) ↗

Top: Done + site title above small URL with padlock + reload/share. Bottom: back/forward/share/tabs/Safari-handoff. App can tint the bar but not restructure. iOS 14+ adds collapse-on-scroll. Reader mode and AutoFill come free.

Baseline truth. Anything heavier in a host app is a deliberate (often costly) choice.

no shot yet
Chrome Custom Tabs (Android) ↗

Single top bar: × + site title (or URL) + optional brand action button + ⋯. Toolbar color themable per scheme. setUrlBarHidingEnabled() collapses bar on scroll. Picture-in-picture added 2024.

Cleaner than iOS equivalent — single bar, hides on scroll. Strong baseline for "good default."
